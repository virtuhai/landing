# API and data integrations

The six-month prototype plans to research and integrate selected capabilities from these services, subject to their current terms, technical limits, availability and attribution requirements:

| Service | Planned use |
| --- | --- |
| OpenSky Network | Airspace and aircraft-related situational context. |
| OpenStreetMap + Overpass API | Geographic context, features and infrastructure queries. |
| GDELT | Public-information and media-event context. |
| NASA FIRMS | Fire and thermal-anomaly context. |

Each adapter will isolate external dependencies, record source and retrieval time where possible, manage rate limits and degrade safely when data is unavailable. The project will not claim real-time coverage, completeness or official status beyond what a source provides.

# API та інтеграції даних

У межах шестимісячного прототипу планується дослідити та інтегрувати вибрані можливості таких сервісів — з урахуванням їхніх актуальних умов, технічних обмежень, доступності та вимог до атрибуції:

| Сервіс | Заплановане використання |
| --- | --- |
| OpenSky Network | Контекст повітряного простору та інформація, пов’язана з повітряними суднами. |
| OpenStreetMap + Overpass API | Географічний контекст, об’єкти та запити до інфраструктури. |
| GDELT | Контекст публічної інформації та медіаподій. |
| NASA FIRMS | Контекст пожеж і теплових аномалій. |

Кожен адаптер ізолюватиме зовнішню залежність, за можливості фіксуватиме джерело й час отримання, керуватиме лімітами запитів і безпечно деградуватиме за відсутності даних. Проєкт не заявлятиме про покриття в реальному часі, повноту або офіційний статус понад те, що надає джерело.
